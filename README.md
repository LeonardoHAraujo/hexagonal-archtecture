> Initial API example using Node.js + TypeScript with OOP and hexagonal archtecture.

## Required global dependencies

- Node.js v16
- Yarn v1

## Up & Running

1. Install local dependencies:

```terminal
yarn
```

2. Rename env:

```terminal
mv .env.example .env
```

3. Get the server up:

```terminal
yarn dev
```

| `src/domain/repositories`     | Are the interfaces/namespaces of the repositories.|

| `src/ports`                   | This is the layer responsible for connecting the adapters to the domain's business rules.|
| `src/ports/db`                | Here is the implementation of database repositories that the domain rules require.|
| `src/ports/http`              | Here is the factories that communicate with the domain and return to the adapter.|
| `src/adapters`                | This is a external world.|
| `src/adapters/prisma`         | Are the service from external world, in this folder have helpers and main core from the database library.|
| `src/adapters/express`        | They are services from the outside world too, in this folder there is the use case factories.|

## Important usage information

### Rules of archtecture

- The `domain` of the application does not know anyone from the outside world, they can only talk to each other.
- The `ports` layer is what communicates the `domain` layer with the `adapters` layer, how it works:
    - In the `ports` layer, they have folders that correspond to the implementations that relate the `domain` layer and the `adapters` layer. Also have app ports entrypoints if needed;
    - The `ports` layer relates to both the `domain` layer and the `adapters` layer;
    - Folders in the `ports` layer should contain generic names that relate to the service adapters. Example:
        - `src/ports/http -> src/adapters/express`;
        - `src/ports/db -> src/adapters/prisma`;
- The `adapters` layer is the outputs to the outside world, any new service from the outside world that is needed can be applied there, but this layer only communicates with the `ports` layer, never directly with the `domain`. how it works:
    - The `prisma` for example is a database layer that communicates with the outside world, it can contain helpers and configurations to serve the `ports` layer;
    - The `express` for example, has its necessary configuration, exports the start of the application and has its endpoints that consume the existing use case factories of the `ports` layer;
